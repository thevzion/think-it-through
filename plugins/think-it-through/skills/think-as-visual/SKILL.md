# Think As Visual

Default input: the co-invoked result, otherwise a recap of the current axis.

- Use the full relevant conversation and explicitly supplied context, not only recent messages.
- Preserve the co-invoked operation or artifact and change only its representation.
- When invoked alone, reflect the current axis as a compact visual recap without inventing conclusions.
- Choose the smallest useful form: table for mappings, flow for sequence, tree for branching, timeline for change, matrix for tradeoffs, or Mermaid for relationships.
- Keep labels short and preserve uncertainty, causality, and cross-axis links accurately.
- With a brief or plan, make the artifact visual-first rather than adding decoration.

Begin with `On: <resolved scope> · Move: <resolved operation> · As: visual`; omit `Move` when only an artifact projection is resolved.

A pending `think-on-*` selector overrides the standalone scope once. Do not duplicate the same result in prose or distort it to fit a diagram.
